html에서 자스 포함시킬 때 어떻게 포함시키는 것이 효율적일까? async vs defer parsing == 분석하다 브라우저는 html을 한줄씩 파싱한다. 스크립트 보이면 서버에서 해당 파일 다운받고 넘어감 -> 느리면 사용자한테도 늦게 전달됨 -> 스크립트를 헤드에 포함시키는 것은 좋지 않다 SO, 바디 마지막에 넣는 방법 FETCHING : 서버에서 받아온다 마지막에 서버에서 패칭함 -> 사용자는 기본 HTML은 빨리 보지만 유의미한 컨텐츠를 보기에는 좀 느림 So then? head + async 사용 async를 발견하면 브라우저는 병렬로 소스를 다운받는다 -> 바디끝에 넣는 것보다 시간이 절약된다 -> 병렬이라 원하는 데이터가 먼저 나오지 않을 수도 있다, 중간에 멈출 수도 있음(blocked) So what? head + defer 파싱하다가 스크립트 디퍼를 발견하면 일단 다운받으라 명령하고 html 로딩 후 js 보여준다. -> 가장 좋아보여!
